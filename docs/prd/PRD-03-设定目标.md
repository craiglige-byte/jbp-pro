# PRD-03：设定目标

> JBP Pro — 联合生意规划助手 · 模块 3/7
>
> 对应组件：`components/ObjectiveStep.tsx`
> 对应数据分区：`JBPData.objectives[]`、`authorizationConfirmed`、`authorizationPolygons`
> 对应 WizardStep：`'objectives'`

---

## 1. 模块概述

设定目标（G - Goals）是 JBP 规划的第三步，也是 GSMT 框架的起点。本模块包含 **4 个必选目标设定**。授权地图范围确认已移至经营回顾页面。4 个目标为固定不可删除项，分别是：达成进货承诺、实现销售目标、守住库存健康、提升盈利能力。

---

## 2. 核心功能清单

| 序号 | 功能 | 说明 |
|------|------|------|
| F01 | 授权地图范围确认 | 高德地图展示 + 多边形编辑（合并/分割/删除/重置）+ 确认按钮 |
| F02 | 达成进货承诺 | 文本模板输入框，输入金额自动联动销售目标 |

## 3. 数据模型

### 3.1 目标数据结构

```typescript
interface JBPObjective {
  id: string;                    // 唯一 ID，如 'obj_1'
  title: string;                 // 目标标题（固定 4 选 1）
  targetValue: string;           // 目标模板文本（含格式化数字）
  metric?: string;               // 指标类型，如 "KPI"
  strategies: JBPStrategy[];     // 策略列表（下一步填写）
  keyResults?: JBPKeyResult[];   // 关键结果（非 4 个特殊目标时使用）
  monthlyTargets?: Record<string, number>;
  lastYearMonthlyTargets?: Record<string, number>;
  // 4 个拆解计划（下一步完成）
  purchasePlan?: JBPPurchasePlan;
  salesPlan?: JBPSalesPlan;
  inventoryPlan?: JBPInventoryPlan;
  profitabilityPlan?: JBPProfitabilityPlan;
}
```

### 3.2 4 个固定目标模板

| 目标 | 输入字段 | 输入格式 | 联动规则 |
|------|---------|---------|---------|
| 达成进货承诺 | 金额（万元） | `¥{金额}` → 格式化中文大写 | 自动计算销售目标：箱数=金额÷45，金额=金额÷0.81 |
| 实现销售目标 | 箱数 | `{箱数} 箱` → 自动计算金额 | 金额=箱数×(45/0.81) |
| 守住库存健康 | 天数 | `≤ {天数} 天` | 无联动 |
| 提升盈利能力 | 百分比 | `{百分比}%` | 无联动 |

### 3.3 目标模板文本示例

**达成进货承诺**：
```
为保障市场供应并深化战略协作，我司承诺在 元气森林2027财年（2026年12月1日至2027年11月30日） 内，
根据双方共同确认的滚动预测，完成总计 人民币壹佰贰拾捌万元整（¥1,280,000） 的 Sell-in进货。
该目标旨在确保核心产品对渠道的充足供应，为终端销售增长奠定基础。
```

**实现销售目标**：
```
为共同扩大市场份额，我司承诺在 元气森林2027财年（2026年12月1日至2027年11月30日） 内，
通过全渠道精细化运营，达成终端市场 Sell-out 销售目标 28,444 箱（约合人民币 1,580,247 元），
并提升核心单品在辖区的渗透率。
```

**守住库存健康**：
```
为提升资金使用效率、保障产品新鲜度并实现可持续增长，我司承诺在 元气森林2027财年 内，
将所经销元气森林产品的平均库存周转天数优化至 ≤ 35 天。
通过精细化的进销存协同管理，实现更敏捷的市场响应。
```

**提升盈利能力**：
```
为确保生意长期健康发展，品牌方与我司协力，在 元气森林2027财年 内，
通过共推高毛利产品、共投精准营销与共管运营效率，将联合生意的年度经营利润率提升至 19%，
实现有质量的增长。
```

## 5. 交互流程

### 5.1 主流程

```
进入页面
  ↓
授权范围确认
  ├─ 未确认：查看地图 → [调整授权区域] → 编辑多边形 → [保存修改] → [确认范围]
  └─ 已确认：显示「范围已确认」绿色标记
  ↓
填写 4 个目标
  ① 进货承诺：输入金额 → 自动联动计算销售目标
  ② 销售目标：输入箱数 → 自动计算金额（也可独立编辑）
  ③ 库存健康：输入目标天数
  ④ 盈利能力：输入目标利润率
  ↓
点击「下一步：策略与衡量」
  ├─ 校验通过 → 导航到 strategies
  └─ 校验失败 → 顶部浮层提示具体未完成的项
```

### 5.2 进货→销售联动规则

```
用户输入进货金额 N（万元）
  ├─ 计算销售箱数：Math.round(N × 10000 / 45)
  │   → 写入「实现销售目标」的 targetValue
  └─ 计算销售金额：Math.round(N × 10000 / 0.81)
      → 写入「实现销售目标」的辅助金额显示

用户独立编辑销售箱数
  └─ 计算销售金额：Math.round(箱数 × 45 / 0.81)
      → 更新金额显示，不影响进货目标
```

## 6. 表单校验规则

### 6.1 校验条件

```typescript
const validateObjectives = (): boolean => {
  // 条件 1：授权范围已确认
  if (!data.authorizationConfirmed) → 提示「请确认授权地图范围」

  // 条件 2：4 个目标均填写
  for (obj of ['达成进货承诺','实现销售目标','守住库存健康','提升盈利能力']) {
    if (!obj.targetValue?.trim()) → 提示「请填写"{目标名}"」
  }

  // 条件 3：「守住库存健康」含有效天数
  if (!obj.targetValue.match(/≤\s*(\d+)\s*天/)) → 提示「请填写天数」

  // 条件 4：「提升盈利能力」含有效百分比
  if (!obj.targetValue.match(/提升至\s*(.*?)\s*%/)) → 提示「请填写百分比」

  // 条件 5：「实现销售目标」含有效箱数
  if (!obj.targetValue.match(/销售目标\s*([\d,]+)\s*箱/)) → 提示「请填写箱数」
}
```

## 9. 边界与异常

| 场景 | 处理方式 |
|------|---------|
| 地图 SDK 未加载 | 动态加载 + 回调初始化 + isMounted 保护 |
| 合并的两块多边形不相邻 | Turf.js 合并失败 → 弹窗提示 |
| 分割线未完全穿过多边形 | 分割失败 → 弹窗提示正确画法 |
| 用户取消编辑 | 地图恢复编辑前状态，确认状态保持不变 |
| 保存编辑后 | 多边形坐标持久化，确认状态重置为 false |
| 删除确认 | 自定义 React 弹窗（避开高德地图 confirm bug） |
