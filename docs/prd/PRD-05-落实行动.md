## 1. 模块概述

落实行动（T - Tactics）是 JBP 规划的第五步，将上一步的策略转化为具体的、可追踪的行动计划。本模块提供两种视图：**列表视图**（按目标→策略→行动三层展开）和**季度排期视图**（拖拽式甘特图）。

### 1.1 核心功能清单

| 序号 | 功能 | 说明 |
|------|------|------|
| F01 | 列表视图 | 目标→策略→行动三层展开/折叠，支持添加/编辑/删除行动项 |
| F02 | 季度排期视图 | 左侧待安排任务栏 + 右侧 Q1~Q4 四列拖拽矩阵 |
| F03 | 拖拽排程 | 拖拽行动项到季度列区域完成时间分配 |
| F04 | 行动项编辑 | 标题、内容(描述)、负责人(多选)、截止日期(季度/自定义) |
| F05 | 负责人管理 | 10 个预设角色 + 自定义角色输入，多选，颜色标签 |
| F06 | 视图切换 | 列表视图 ↔ 季度排期一键切换 |
| F07 | 按负责人筛选 | 季度排期视图支持按角色过滤 |
| F08 | 旧数据迁移 | 自动将旧版 `owner: string` 迁移为 `owners: string[]` |

## 2. 数据模型

### 2.1 行动项结构

```typescript
interface JBPAction {
  id: string;           // 唯一 ID
  title?: string;       // 行动标题（与 content 分离展示）
  text: string;         // 行动文本（兼容旧格式，与 content 互备）
  content?: string;     // 行动详细内容
  owners: string[];     // 负责人列表（多人）
  deadline: string;     // 截止时间：Q1/Q2/Q3/Q4 或自定义字符串（如「每月」）
  status: 'pending' | 'in_progress' | 'completed';
  startMonth?: string;
  endMonth?: string;
  scheduledMonth?: string;
}
```

## 4. 交互流程

### 4.1 列表视图操作

```
添加行动：
  点击 [+ 添加行动] → 创建空行动项 → 自动聚焦标题输入
  ↓
编辑标题：点击标题 input → 输入
  ↓
编辑内容：点击内容 textarea → 输入（2 行自适应高度）
  ↓
添加负责人：
  点击 [+ 添加负责人] → 下拉面板
  ├─ 自定义输入框 → Enter 添加
  └─ 预设角色列表（可多选，已选显示 ✓）
  ↓
设置排期：
  下拉选择 Q1/Q2/Q3/Q4 → 或选「自定义」
  ├─ 自定义模式：Q1~Q4 多选复选框 + 自由文本输入
  └─ 如输入「每月」「12月31日前」等
  ↓
删除行动：点击 🗑️ → 立即删除
```

### 4.2 季度排期操作

```
拖拽排程：
  1. 左侧待安排栏中，按住任务卡片
  2. 拖到右侧对应季度列
  3. 松开完成排程 → deadline 更新为该季度
  ↓
拖回待安排：
  1. 从季度列拖回左侧待安排栏
  2. 松开 → deadline 清空
  ↓
跨季度移动：
  1. 从 Q1 列拖到 Q2 列 → deadline 更新为 Q2
```

### 4.3 下一步校验

```
点击「下一步」
  ↓
getFirstIncompleteItem()
  ├─ 某策略无行动项 → 提示「请为"XX"下的策略"YY"添加行动项」
  ├─ 某行动项不完整 → 提示缺少的具体字段（标题/内容/负责人/排期）
  └─ 全部完成 → 导航到 budget
```

## 5. 导航规则

- 进入条件：`isStrategyStepComplete()` → 所有策略 text+measure 非空
- 离开条件：`isActionStepComplete()` → 每个策略至少 1 个行动项 + 每个行动项四字段完整
- 上一步：`'strategies'`
- 下一步：`'budget'`

### 5.1 行动项完整性判定

```typescript
const isActionComplete = (action: JBPAction): boolean => {
  return !!(
    action.title?.trim() &&                           // 有标题
    (action.content?.trim() || action.text?.trim()) && // 有内容
    action.owners?.length > 0 &&                       // 有负责人
    action.deadline?.trim()                            // 有排期
  );
};
```

## 6. 自动填充机制

从 StrategyStep 选择预置策略时，自动生成的行动项结构示例：

```
策略「资金前置储备」
  ├─ 行动1: 资金需求测算与来源确认
  │   负责人: 经销商老板
  │   排期: 未排期
  │   内容: 测算30%储备金额，并明确资金来源...
  │
  └─ 行动2: 资金存入专用账户
      负责人: 经销商老板
      排期: 未排期
      内容: 由经销商老板在12月31日前...
```

策略「人效提升与激励绑定」
  ├─ 行动1: 人与片区匹配优化 → 销售主管
  ├─ 行动2: 绩效奖金方案设计 → 经销商老板
  ├─ 行动3: 月度排名与即时激励 → 销售主管
  └─ 行动4: 协访帮扶机制 → 销售主管

## 7. 边界与异常

| 场景 | 处理方式 |
|------|---------|
| 某策略无行动项但用户尝试下一步 | 展开对应目标 → 高亮策略行 → 顶部提示缺少行动项 |
| 行动项缺少字段 | 红框高亮缺失字段 → 星号标记 → 顶部提示具体缺失项 |
| 旧版 owner 数据迁移 | useEffect 自动检测并转换 `owner` → `owners[]` |
| 负责人下拉面板外部点击 | 点击其他区域自动关闭 |
| 拖拽未完成就释放 | 还原到原位置 |
| 所有任务已安排完毕 | 待安排栏显示绿色 ✔️「所有任务已安排完毕」 |
