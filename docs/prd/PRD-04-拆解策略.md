# PRD-04：拆解策略

> JBP Pro — 联合生意规划助手 · 模块 4/7
>
> 对应组件：`components/StrategyStep.tsx` + 4 个 Breakdown 面板
> 对应数据分区：`JBPData.objectives[].strategies[]` + `purchasePlan` / `salesPlan` / `inventoryPlan` / `profitabilityPlan`
> 对应 WizardStep：`'strategies'`

---

## 1. 模块概述

拆解策略（S&M - Strategies & Measures）是 JBP 规划的第四步。本模块支持两个版本：

| 版本 | 步骤名称 | 策略功能 | 落实行动 |
|------|---------|---------|---------|
| ≥500万 | 拆解策略 | 有策略库+策略输入 | 有 |
| <500万 | 拆解目标 | 无策略 | 无（整步跳过） |

每个目标需要先完成「目标拆解」（即把大目标分解为可执行的子计划），≥500万版本再匹配「策略与衡量标准」。

### 1.1 核心功能清单

| 序号 | 功能 | 说明 |
|------|------|------|
| F01 | 进货拆解面板 | 品类/季度拆分 + 季度×品类矩阵 + 月度权重 + 场景规划 |
| F02 | 销售拆解面板 | 时间维度拆解 + 人员维度拆解 + 月度区域交叉目标表 |
| F03 | 库存拆解面板 | 品类库存设置 + 月度库存规划 + 安全库存预警 |
| F04 | 盈利拆解面板 | 收入/成本/费用测算 + 费用大类拆解 |
| F05 | 策略选择与填写 | 参考策略库（纯展示）+ 自定义策略输入（仅策略文本，无衡量标准）|
| F06 | 逐步引导验证 | 检测首个未完成的部分→滚动+高亮→提示用户完成 |
| F07 | 联动计算 | 库存拆解依赖进货计划和销售计划数据 |
| F08 | 策略自动填充行劢 | 选择策略时自动生成对应的预制行动项 |

## 2. 子模块详细规格

### 2.1 进货拆解 — PurchaseBreakdown

**数据分区**：`JBPData.objectives[达成进货承诺].purchasePlan`

**拆解层级**：

```
          年度进货总目标
              │
    ┌─────────┼─────────┐
    ▼         ▼         ▼
 品类拆分   季度拆分   月度规划
 (6品类)    (Q1~Q4)   (12个月)
    │         │         │
    └────┬────┘         │
         ▼              │
   季度×品类矩阵        │
         │              │
         └──────────────┤
                        ▼
                  月度场景规划
              (场景/逻辑/占比/总额)
```

**品类拆分表**：

| 品类 | 占比(%) | 金额(万元) |
|------|---------|-----------|
| 电解质水 | 35 | 448 |
| 气泡水 | 25 | 320 |
| 冰茶 | 20 | 256 |
| 维生素水 | 10 | 128 |
| 好自在水 | 7 | 90 |
| 其他 | 3 | 38 |

**季度拆分表**：

| 季度 | 占比(%) | 金额(万元) |
|------|---------|-----------|
| Q1 | 22 | 282 |
| Q2 | 28 | 358 |
| Q3 | 30 | 384 |
| Q4 | 20 | 256 |

**季度×品类矩阵**：6×4 交叉表，展示每个品类在每个季度的金额分配。

**月度场景规划表**：

| 月份 | 场景 | 逻辑 | 占比(%) | 总额 | 品类分解值 |
|------|------|------|---------|------|-----------|
| 2025-12 | 🧧年货 | 春节备货 | 12 | XX | 各品类×12% |
| ... | ... | ... | ... | ... | ... |

**按钮**：无拆解方案时显示「目标拆解」，已有方案时显示「调整拆解」。

**拆解向导（4步骤）**：

| 步骤 | 名称 | 校验规则 | 不满足时按钮文案 |
|------|------|---------|-----------------|
| Step 1 | 年度拆解 | 品类占比（整数）+ 季度占比（整数）各自和=100% | 请调整至100% |
| Step 2 | 季度品类 | 每个品类4季度%和=100%（百分比一位小数，金额整数） | 请将各品类调节至100% |
| Step 3 | 月度权重 | 每季度3个月%和=100%（一位小数） | 请将各季度拆解至100% |
| Step 4 | 确认计划 | 无校验 | 确认生成计划 |

**输入限制**：Step1 品类/季度占比仅允许非负整数；Step2 金额仅允许非负整数、百分比仅允许非负一位小数；Step3 月度权重仅允许非负一位小数。默认值全部为 0，需用户自行填写。

Top bar 显示 `¥{totalTarget}`（非 `xx万`），Current 也显示 `¥xx.xx`。

### 2.2 销售拆解 — SalesBreakdown

**按钮**：无拆解方案时显示「目标拆解」，已有方案时显示「调整拆解」。

**拆解向导（3步骤）**：

**数据分区**：`JBPData.objectives[实现销售目标].salesPlan`

**拆解维度**：

**① 时间维度拆解**：

| 月份 | 去年实绩 | 去年占比 | 场景 | 今年占比 | 今年目标 |
|------|---------|---------|------|---------|---------|
| 12月 | 85 | 7.4% | 春节备货 | 8% | 2,276 |
| 1月 | 142 | 12.4% | 春节旺季 | 13% | 3,698 |
| ... | ... | ... | ... | ... | ... |

**② 人员维度拆解**：

| 区域 | 去年负责人 | 去年实绩 | 去年占比 | 场景 | 今年占比 | 今年目标 | 今年负责人 |
|------|-----------|---------|---------|------|---------|---------|-----------|
| 城中 | 张经理 | 350 | 30% | 存量稳增 | 28% | X | 张经理 |
| 城东 | 李主管 | 280 | 24% | 存量稳增 | 22% | X | 李主管 |
| ... | ... | ... | ... | ... | ... | ... | ... |

**③ 月度×区域交叉目标表**：12×N 矩阵，行列交叉处为具体目标值。

**拆解向导校验**：

| 步骤 | 名称 | 校验规则 | 不满足时按钮文案 |
|------|------|---------|-----------------|
| Step 1 | 时间节奏拆解 | 12个月今年占比和=100% | 请按月将今年目标拆解至100% |
| Step 2 | 人员/片区拆解 | 各片区今年占比和=100% | 请按人员/片区拆解至100% |
| Step 3 | 确认计划 | 季度=各月汇总，片区=月×人员% | 确认并保存 |

**输入限制**：今年设定占比仅允许非负一位小数，默认值为 0，场景文案保留默认值。

### 2.3 库存拆解 — InventoryBreakdown

**数据分区**：`JBPData.objectives[守住库存健康].inventoryPlan`

**依赖关系**：依赖「达成进货承诺」的 purchasePlan 和「实现销售目标」的 salesPlan

**品类库存设置表**：

| 品类 | 周转天数 | 占比(%) | 理由 |
|------|---------|---------|------|
| 电解质水 | 25 天 | 30% | 高动销，快速周转 |
| 气泡水 | 30 天 | 25% | 中等动销 |
| 冰茶 | 35 天 | 20% | 季节性波动 |
| 维生素水 | 28 天 | 12% | 新兴品类，适度备货 |
| 好自在水 | 40 天 | 8% | 长尾品类 |
| 其他 | 45 天 | 5% | 低动销 |

**月度库存规划表**：

| 月份 | 场景 | 销售占比(%) | 销量预估(箱) | 品类库存值 | 总库存 |
|------|------|------------|-------------|-----------|--------|
| 12月 | 春节备货 | 12 | X | 品类A: X, 品类B: X | X |

**拆解向导（2步骤）**：Step 1「品类周转策略」（设置周转天数+策略备注），Step 2「月度库存推演」（含策略说明蓝色提示框）。Step2 的「销量预估(箱)」为只读字段，数据取自销售目标拆解→时间节奏拆解→明年预估销售箱数（全司合计），按月份同月匹配。

**月份显示**：12月~11月（财年顺序），使用 shortLabel（无年份）。

### 2.4 盈利拆解 — ProfitabilityBreakdown

**数据分区**：`JBPData.objectives[提升盈利能力].profitabilityPlan`

**拆解结构**：

```
销售收入 (Sales Revenue)
+ 返利收入 (Rebate Revenue)
- COGS (成本)
= 毛利
- 运营费用 (Operating Expenses)
= 净利润
→ 目标利润率 (Target Profit Margin)
```

**费用大类拆解表**：

| 费用类别 | 费用项 | 去年占比 | 去年实绩 | 今年目标 | 今年占比 | 策略 |
|---------|--------|---------|---------|---------|---------|------|
| 人员费用 | 基本工资 | ... | ... | ... | ... | ... |
|  | 绩效奖金 | ... | ... | ... | ... | ... |
| 物流费用 | 仓库租金 | ... | ... | ... | ... | ... |
|  | 车辆费用 | ... | ... | ... | ... | ... |
| 市场费用 | 陈列费 | ... | ... | ... | ... | ... |
|  | 促销费 | ... | ... | ... | ... | ... |
| 管理费用 | 行政开支 | ... | ... | ... | ... | ... |

**拆解向导（3步骤）**：Step 1「测算运营费用」（列：项目/去年实际/今年目标/备注），Step 2「拆解运营费用」（列：费用细项/去年实际/去年占比/今年目标/今年占比/管控策略），Step 3「确认计划」。

**Target Margin**：顶部显示固定来自 Step 3 设定目标「提升盈利能力」填写的百分比值（默认 7%），不随 Step 1 输入变化。表内净利润率动态计算。

**年号标签**：采用相对年份标签「今年实际/今年占比/明年目标/明年占比」，不随财年滚动变化，无需每年手动更新。

**计算逻辑**：
- `totalRevenue = salesRevenue + rebateRevenue`
- `targetNetProfit = totalRevenue × targetMargin%`（targetMargin 来自设定目标）
- `maxOperatingExpenses = totalRevenue - cogs - targetNetProfit`

### 2.5 策略与衡量（仅≥500万版本）

**左侧：参考策略库**

从 `OBJECTIVE_TEMPLATES` 加载预置策略列表（只读参考）：

| 目标 | 预置策略数 | 标签体系 |
|------|-----------|---------|
| 达成进货承诺 | 3 | 资金、管理、运营 |
| 实现销售目标 | 5 | 团队、产品、渠道、渠道、管理 |
| 守住库存健康 | 5 | 供应链、数据、运营、管理、考核 |
| 提升盈利能力 | 6 | 产品、渠道、财务、运营、管理、财务 |

**右侧：已选策略**

每条策略包含：
- 策略文本（textarea，必填）
- 衡量标准 M:（input，必填）
- 标签（自动从模板继承）
- 贡献度（4 个特殊目标可选，普通目标推荐填写）
- 删除按钮

**自定义策略输入**：底部输入框 + Enter 键 / 添加按钮

**自动填充行动项**：选择预置策略时，自动生成 2~5 条预制行动项（根据策略类型差异化），包含标题、内容、负责人、截止日期、状态

## 3. 逐步引导验证

### 3.1 优先顺序

每个目标的完成优先级：
1. **先完成目标拆解**（purchasePlan / salesPlan / inventoryPlan / profitabilityPlan）
2. **再添加策略**（至少 1 条）
3. **最后完善策略内容**（text + measure 均非空）

### 3.2 引导行为

```
用户点击「下一步」
  ↓
getNextIncompletePart() 遍历所有目标
  ├─ 找到第一个未完成拆解的目标
  │   → 滚动到该目标位置
  │   → 高亮 ring-2 ring-amber-400
  │   → 显示顶部 Toast：「请先完成"XX目标"的目标拆解」
  │
  ├─ 所有拆解完成，找第一个未添加策略的目标
  │   → 高亮策略区域红框
  │   → 显示 Toast：「请先为"XX目标"添加策略」
  │
  └─ 所有完成 → 允许导航
```

## 5. 导航规则

- 进入条件：`data.objectives.length > 0`
- 离开条件：`isStrategyStepComplete()` → 所有目标拆解完成 + 所有策略 text+measure 非空
- 上一步：`'objectives'`
- 下一步：`'actions'`
