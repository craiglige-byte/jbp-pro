## 会话拆解原则

本文档使用以下拆解原则：

| 原则 | 说明 |
|------|------|
| **页面闭环** | 每个独立步骤页 + 其子面板/弹窗自成会话 |
| **共享底座优先** | 类型定义、共享组件、导航框架先做或先对齐 |
| **复杂子模块独立** | 经营回顾的 10 个子模块各自可独立迭代 |
| **同构面板批量复用** | 拆解策略的 4 个子拆解面板复用同一套 Wizard/Breakdown 底座 |
| **数据流闭环** | 每个模块从 `JBPData` 中读取自己的数据分区，互不侵入 |

### 粒度判断（一箭之遥测试）

> 一个会话模块的内容，能否在一次对话里完成以下 5 件事？
> 1. 数据模型对齐（types.ts 相关 interface）
> 2. UI 组件结构（面板 + 表单 + 图表）
> 3. 共享组件复用（图表底座、表格底座、Wizard 框架）
> 4. 核心交互跑通（填写 → 保存 → 下一步）
> 5. 文档同步（本拆解文件状态更新）

- **能** → 粒度合适
- **不能** → 考虑再拆一刀（子模块独立成会话）
- **太轻松** → 考虑合并相邻模块

## 推荐会话总览

### A. 项目底座（跨模块共享）

#### SESSION-SHARED-01：类型系统与数据模型对齐
- **现有文件**：`types.ts`、`constants.ts`
- **数据分区**：`JBPData`（全局）、所有子类型 interface
- **范围**：确保所有 7 个步骤的数据结构一致、无遗漏字段、无类型冲突
- **依赖**：无（最优先）

#### SESSION-SHARED-02：步骤导航框架与路由
- **现有文件**：`App.tsx`、`components/StepWizard.tsx`
- **范围**：7 步 Wizard 导航、前进/后退权限判定、步骤完成状态检测、localStorage 持久化
- **依赖**：SESSION-SHARED-01

## 推荐执行顺序

```
SESSION-SHARED-01：类型系统与数据模型对齐          ← 最先
    ↓
SESSION-SHARED-02：步骤导航框架与路由
    ↓
SESSION-INFO-01：基本信息页
    ↓
┌───────────────────────────────────────────────┐
│  经营回顾子模块（可并行推进）                      │
│  SESSION-REVIEW-01 ~ SESSION-REVIEW-10         │
│  （子模块间无强依赖，数据分区独立）                 │
│  ↓                                             │
│  SESSION-REVIEW-00：经营回顾汇总页               │
└───────────────────────────────────────────────┘
    ↓
SESSION-OBJECTIVE-01：目标设定页
    ↓
SESSION-STRATEGY-01：策略总览页
    ↓
┌───────────────────────────────────────────────┐
│  策略拆解子模块（可并行推进）                      │
│  SESSION-STRATEGY-02：进货拆解                   │
│  SESSION-STRATEGY-03：销售拆解                   │
│  SESSION-STRATEGY-04：库存拆解                   │
│  SESSION-STRATEGY-05：盈利拆解                   │
└───────────────────────────────────────────────┘
    ↓
SESSION-ACTION-01：行动战术页
    ↓
SESSION-BUDGET-01：预算总览页
    ↓
┌───────────────────────────────────────────────┐
│  预算子模块（可并行推进）                         │
│  SESSION-BUDGET-02：仓库租赁                    │
│  SESSION-BUDGET-03：车辆配置                    │
│  SESSION-BUDGET-04：人员架构                    │
│  SESSION-BUDGET-05：费用规划                    │
│  SESSION-BUDGET-06：资金准备                    │
└───────────────────────────────────────────────┘
    ↓
SESSION-REVIEWPLAN-01：计划预览页                  ← 最后
```

### 排序理由

1. **共享底座最优先**：类型系统和导航框架是所有步骤的依赖，必须先稳定
2. **基本信息其次**：经销商名称、周期、授权区域是后续所有分析的基础上下文
3. **经营回顾 → 目标 → 策略 → 行动 → 预算 → 预览**：这是 JBP 业务逻辑的自然递进链，前一步的输出是后一步的输入
4. **经营回顾子模块可并行**：10 个子模块数据分区独立（`operations` / `performance` / `trends` / `productCategories` / ...），无互相写入依赖
5. **策略拆解子模块可并行**：4 个拆解方向共用 `objectives[].strategies[]` 但各有独立的 `xxxPlan` 子分区
6. **行动依赖策略完成**：每个行动项挂在具体策略下，策略必须先确定
7. **预算依赖行动完成**：费用和资源配置需要在行动方案明确后才有依据
8. **预览最后**：需要所有数据就绪后做全量回看和导出
